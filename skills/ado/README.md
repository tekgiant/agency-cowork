# ADO (Azure DevOps)

General-purpose Azure DevOps work item query, search, and update skill. Supports WIQL search, saved queries, filtered listing, and write operations including create, assign, comment, set-field, reparent, and link.

## Features

- WIQL keyword search across work items
- Saved query execution
- Filter by state, type, DRI, iteration, area, or tag
- Create new work items
- Assign or reassign work items
- Add comments to work items
- Set individual fields on work items
- Reparent items in the hierarchy
- Link work items together

## Usage

Trigger phrases: "ADO", "work item", "query ADO", "search ADO", "create work item", "assign work item", "ADO comment", "WIQL".

Examples:
- "Search ADO for bugs assigned to me"
- "Create a new task in the current sprint"
- "Add a comment to work item 12345"

## Prerequisites

- Azure DevOps organization access configured
- ADO personal access token or authentication configured
